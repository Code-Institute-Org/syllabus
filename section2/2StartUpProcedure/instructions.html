    Moving the arm.

    Let us start by looking at the python file. It is divided into four parts:

    1. The first part of the code contains some imports, constant initialisations and some Webots specific code. Webots is the name of the simulation software.
    2. Then, a blank area where you will write your code.
    3. The next section contains code to verify the variables are correctly defined.
    4. The final code section detects if a can is approaching the gripper, grasps it and moves it to the box.

    Now it’s your opportunity to write some code! When looking at the simulation, it should be evident that the manipulator is not in the correct position to grab cans. To fix this, we need to activate the rotational motor that controls the shoulder joint of the robot. To do this, you will need to write two commands:

    shoulder_motor = robot.getDevice('shoulder_pan_joint')
shoulder_motor.setPosition(0.2)

    The first command sets the specific manipulator arm joint we want to move as a variable.

    1. The variable robot is an instance of the class Robot, a blueprint or plan of the real-life UR5e robotic arm object.
    2. The getDevice method of the robot allows us to retrieve a motor or sensor by its name (shoulder_pan_joint).
    3. In the second command, we control the motor's position with another Class method. As it is a rotational motor, we give it an angle value of 0.2 radians. A radian is the standard international unit for measuring angles and it is what the robot controller expects to be used.

    Enable the distance sensor

    The gripper does not close on the cans as the robot uses a distance sensor to detect the cans. The code that links the value returned by the sensor to the gripper motors is written for you. However, you will need to turn the sensor on for it to work.

    First, use the getDevice method to retrieve the distance sensor in the same manner as for the shoulder motor. The sensor is called distance_sensor, and you must name the variable distance_sensor, as well.

    Then you need to enable it using the code command: distance_sensor.enable(TIME_STEP)

    Where distance_sensor is the variable in which we put the result of the call to getDevice().

    The enable function takes the sampling period as an argument. The simulation, like a movie, is advancing in tiny steps: a few tens of milliseconds. This is also how a real-world robot sees its environment. Although time is continuous, the robot's clock speed or sensor sample rate will have a set period.

    The code defined this TIME_STEP constant as 32. We will use this constant value as the sampling period of the distance sensor. Therefore, every 32 milliseconds, the distance sensor will check if there is an object in front of it.

    Reduce the maximum velocity of the arm

    The arm moves too quickly to drop the can correctly, so we need to reduce its speed to fix that.

    However, remember that the arm commands have set position rather than speed so far.

    Therefore, we will reduce the maximum velocity of the robot. We are satisfied with the velocity of the gripper, so we will just modify the speed of the arm itself. We can achieve that by fixing the speed of each motor of the arm: 

    for motor in ur_motors:
    motor.setVelocity(speed)

    You do not have to define ur_motors and speed, as it was done for you in the first part of the code.

    Note that we do not modify the velocity of the shoulder joint because we want the manipulator to be in position over the conveyor belt as quickly as possible.
